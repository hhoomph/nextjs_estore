# Consolidated Learnings

This file contains curated, summarized, and actionable insights derived from `raw_reflection_log.md`. It serves as the primary, refined knowledge base for long-term use.

**Purpose:** Store durable, broadly applicable knowledge that significantly impacts future performance, enables more robust understanding, or provides essential project-specific configurations.

**Organization:** Group insights by topic, project, or technology for easy retrieval.

**Maintenance:** When `raw_reflection_log.md` entries are consolidated here, they should be removed from the raw log to keep it focused on recent, unprocessed reflections.

---

## Memory Bank System

**Initialization Process:**
- Memory bank requires two core files: `raw_reflection_log.md` and `consolidated_learnings.md`
- Raw logs capture detailed, timestamped reflections immediately after tasks
- Consolidated learnings distill actionable insights from raw logs
- Maintain clear documentation of the continuous improvement workflow

**Maintenance Workflow:**
- Regularly review raw logs for entries ready for consolidation
- Remove processed entries from raw logs to maintain focus on recent reflections
- Update consolidated learnings with new distilled insights
- Ensure consolidation process runs smoothly without errors

**File Structure Standards:**
- Use consistent formatting with YAML frontmatter for each entry
- Include date, task reference, and clear categorization
- Document both successes and learnings to build comprehensive knowledge base

## Development Workflow
